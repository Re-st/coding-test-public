# 제목: 더 안전하고 일반적인 DP 풀이 (최소값)
- 적용 조건: 현재 풀이의 복잡한 규칙을 검증하기 어렵고, 더 체계적인 풀이가 필요할 때.
- 핵심 아이디어: 두 개의 DP 테이블을 사용합니다. `dp_any[i]`는 0으로 시작 가능한 i개 성냥의 최소 숫자, `dp_min[i]`는 1-9로 시작하는 최소 숫자입니다. `dp_any`를 먼저 채우고, 이를 이용해 `dp_min`을 계산하면 모든 경우를 안전하게 처리할 수 있습니다.
- 복잡도: O(N) (N=최대 성냥 개수. 테이블 사전 계산 후 조회). 문자열 연산 포함 시 `O(N^2)`.
