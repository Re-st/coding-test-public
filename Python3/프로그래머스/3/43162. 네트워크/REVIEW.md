## 한 줄 결론
- (이 문제는) **그래프 탐색(DFS/BFS)** / 복잡도: O(N^2) / 제일 위험한 실수: 현재 노드의 이웃을 탐색하는 반복문 범위 설정 오류

## 더 빠른/더 안전한 풀이(필수)
- 현 접근(미방문 정점에서 탐색 시작)이 최선입니다. 다만, 재귀 DFS로 구현하면 코드가 더 간결해지고 버그 위험이 크게 줄어듭니다.
  - 아이디어: 미방문 정점(i)을 찾을 때마다, 그 정점에서 재귀 DFS를 호출해 연결된 모든 노드를 방문 처리하고 네트워크 개수를 1 증가시킵니다.
  - 복잡도: O(N^2) (동일)
  - 왜 이게 더 좋은지: 교과서적인 구현이라 의도를 파악하기 쉽고, 비표준적인 반복문 범위 같은 잠재적 버그를 원천 차단합니다.

## 내가 지금 틀리기 쉬운 포인트 TOP3
1.  **탐색 범위**: 현재 노드 `a`의 모든 이웃을 검사해야 하는데, `for j in range(i + 1, n)`은 컴포넌트 시작점 `i`보다 큰 인덱스만 봅니다. 그래프가 무방향이라 운 좋게 통과됐을 수 있지만, 논리적으로 틀렸고 잠재적 버그입니다.
2.  **방문 처리 타이밍**: 탐색 시작 노드 `i`에 대한 `visited` 처리가 누락되었습니다. 현재는 외부 `for` 루프 구조 덕에 중복 방문이 없지만, 탐색 로직은 시작 노드를 포함해 연결된 모든 노드의 방문 상태를 책임져야 합니다.
3.  **자료구조 오해**: 변수명이 `queue`지만 실제로는 `list.pop()`을 사용해 스택(DFS)으로 동작합니다. BFS를 의도했다면 `collections.deque`와 `popleft()` 사용이 필수입니다.

## 즉시 수정 액션(최대 4개)
1.  `for j in range(i + 1, n):` -> `for j in range(n):` 으로 수정하여 모든 이웃을 탐색하도록 변경.
2.  `queue = [i]` 바로 위에 `visited[i] = True`를 추가하여 탐색 시작 노드를 즉시 방문 처리.
3.  (가독성) `queue` 변수명을 `stack`으로 변경하여 DFS 의도를 명확히 표현.
4.  `while queue:` 루프 내 `a = queue.pop()` 바로 아래에 `if visited[a]: continue` 추가 (더 안전한 중복 방문 방지).

```REVIEW_EXAMPLE
# 제목: 더 깔끔하고 안전한 재귀 DFS 풀이
- 적용 조건: 코드를 더 표준적이고 이해하기 쉽게 만들고 싶을 때.
- 핵심 아이디어: 외부 루프에서 미방문 노드를 찾고, 발견 시 재귀 함수를 호출해 연결된 모든 노드를 방문 처리하는 구조로 분리합니다.
- 복잡도: O(N^2)
- 구현 스케치(Python):
```python
def solution(n, computers):
    answer = 0
    visited = [False] * n

    def dfs(u):
        visited[u] = True
        for v in range(n):
            # 연결되어 있고 아직 방문하지 않았다면 재귀 호출
            if computers[u][v] == 1 and not visited[v]:
                dfs(v)

    # 모든 노드를 순회하며 확인
    for i in range(n):
        if not visited[i]:
            dfs(i)
            answer += 1  # DFS가 한 번 끝나면 하나의 네트워크(컴포넌트)를 찾은 것
    
    return answer
```
