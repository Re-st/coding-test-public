## 내가 지금 틀리기 쉬운 포인트 TOP3

- 2) 인덱스/경계/오버플로/방문처리: 경로 생성 중 격자 밖으로 나가는 경우를 고려해야 합니다. 현재 코드는 부분적으로만 처리하며, 전체 로직상 허점이 있습니다.
- 3) 자료구조/성능 함정: 현재의 '남은 이동 횟수 소진'과 '목표 도달'을 분리한 접근은 그리디의 핵심(매 순간 최적 선택)을 위반하여 사전 순 최적 경로를 보장하지 못합니다. (예: 예제 1번 `dllrl` 생성 실패)

## 즉시 수정 액션(최대 4개)
- 1. 시작 전, `dist = abs(x-r) + abs(y-c)`를 계산 후 `k < dist` 또는 `(k - dist) % 2 != 0` 이면 즉시 `"impossible"`을 반환하도록 수정하세요.
- 2. `while`과 `if`로 분리된 현재 로직을 `k`번 반복하는 단일 `for` 루프로 변경하세요.
- 3. 루프 내부에서 `d, l, r, u` 순서로 이동 가능성을 검사합니다.
- 4. "이동 가능" 조건: (1) 격자 내부 + (2) 이동 후 남은 횟수가 목적지까지의 맨해튼 거리보다 크거나 같아야 합니다.

```REVIEW_EXAMPLE
# 제목: 더 정확한 그리디 풀이 스케치
- 적용 조건: 항상 현재 풀이보다 이 접근법이 더 정확하고 논리가 간단합니다.
- 핵심 아이디어: `k`번의 이동 기회마다 `d, l, r, u` 순서로 이동을 시도합니다. 해당 이동이 (1)격자 밖을 벗어나지 않고, (2)이동 후 남은 횟수 내에 목적지에 도달 가능한 경우에만 그 방향으로 실제 이동합니다.
- 복잡도: O(k)
- 구현 스케치(선택): 
```python
def solution(n, m, x, y, r, c, k):
    dist = abs(x - r) + abs(y - c)
    if k < dist or (k - dist) % 2 != 0:
        return "impossible"

    answer = ""
    # 사전순: d, l, r, u
    dirs = {'d': (1, 0), 'l': (0, -1), 'r': (0, 1), 'u': (-1, 0)}

    for _ in range(k):
        for move_char in "dlru":
            dx, dy = dirs[move_char]
            nx, ny = x + dx, y + dy

            if 1 <= nx <= n and 1 <= ny <= m:
                rem_k = k - len(answer) - 1
                rem_dist = abs(nx - r) + abs(ny - c)
                if rem_k >= rem_dist:
                    answer += move_char
                    x, y = nx, ny
                    break
    
    return answer
```
