## 내가 지금 틀리기 쉬운 포인트 TOP3
- 1) **역추적 논리**: 현재 `dp` 값과 이웃 `dp` 값을 비교하는 방식도 정답이지만, `dp` 테이블을 채울 때 사용한 조건(`s1[i-1] == s2[j-1]`)을 역추적에 직접 활용하면 로직이 더 명확해집니다.
- 2) **인덱스 관리**: `dp[i][j]`가 `s1[i-1]`과 `s2[j-1]`까지의 결과를 의미하므로, `dp` 인덱스와 문자열 인덱스 간 `-1` 차이를 일관되게 유지하는 것이 중요합니다. 현재 코드는 잘 지키고 있습니다.
- 3) **결과 문자열 조합**: 역추적 시 LCS 문자를 뒤에서부터 찾게 되므로, `string`에 문자를 추가한 뒤 마지막에 `reverse`하는 현재 방식은 정석적이고 올바릅니다.

## 즉시 수정 액션(최대 4개)
- **(선택) 역추적 로직 가독성 개선**: 버그는 아니지만, DP 테이블을 채울 때의 분기문과 유사한 구조로 역추적 로직을 작성하면 코드의 의도가 더 명확해집니다.

  ```cpp
  // 스니펫: 역추적 로직 개선 제안
  int i = n1, j = n2;
  while (i > 0 && j > 0) {
      if (s1[i-1] == s2[j-1]) {         // 1. 문자가 같을 때 (LCS에 포함)
          res += s1[i-1];
          i--; j--;
      } else if (dp[i-1][j] > dp[i][j-1]) { // 2. 위쪽 DP 값이 더 클 때
          i--;
      } else {                            // 3. 왼쪽 DP 값이 더 크거나 같을 때
          j--;
      }
  }
  ```
