## 한 줄 결론
- (이 문제는) **정렬 기반 아이디어** / 복잡도: O(N log N) / 정렬 전후 인덱스 차이 계산 실수

## 더 빠른/더 안전한 풀이(필수)
- 현 접근이 거의 최선입니다. 문제의 핵심은 각 원소가 정렬 후 얼마나 왼쪽으로 이동하는지 파악하는 것이고, 이는 정렬 없이는 알 수 없습니다. O(N log N)이 실질적인 최적입니다.

## 내가 지금 틀리기 쉬운 포인트 TOP3
- 1) 정답 논리: 버블 소트는 정렬이 완료된 **다음** 패스(pass)에서 종료되므로, 필요한 최대 이동 횟수(`max_shift`)에 1을 더한 값이 최종 답입니다.
- 2) 인덱스 계산: 1-based 초기 인덱스와 0-based 최종 인덱스를 혼용하여 `(초기_인덱스 - 최종_인덱스)`를 계산한 결과가 `(이동거리 + 1)`이 되어 정답이 맞았습니다. 이는 혼란을 유발하고 실수하기 쉬운 방식입니다.
- 3) 안정 정렬: 같은 값을 가진 원소들의 상대적 순서 유지가 중요합니다. 직접 작성한 비교 함수도 정확하지만, `pair`의 기본 정렬을 이용하거나 `std::stable_sort`를 쓰는 것이 더 관용적이고 안전합니다.

## 즉시 수정 액션(최대 4개)
- 인덱스를 0-based로 통일하고, `pair`에 `{값, 초기_인덱스}` 순서로 저장하면 코드가 훨씬 명확하고 간결해집니다.
    ```cpp
    vector<pair<int, int>> A(n);
    for(int i = 0; i < n; ++i) {
        cin >> A[i].first; // 값(value)
        A[i].second = i;   // 0-based 초기 인덱스
    }

    sort(A.begin(), A.end()); // pair는 first, second 순으로 자동 정렬

    int max_shift = 0;
    for(int i = 0; i < n; ++i) { // i는 정렬 후 0-based 최종 인덱스
        max_shift = max(max_shift, A[i].second - i);
    }
    cout << max_shift + 1 << '\n';
    ```
