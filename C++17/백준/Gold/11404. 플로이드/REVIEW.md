## 더 빠른/더 안전한 풀이(필수)
- 현 접근이 거의 최선입니다. N(≤100)이 작아 모든 쌍 최단 경로를 구하는 데 O(N³) 플로이드-워셜 알고리즘이 가장 표준적이고 효율적입니다.

## 내가 지금 틀리기 쉬운 포인트 TOP3
- 1) 초기값/시작점/종료조건: `INT_MAX`를 무한대로 사용 시 `A[i][k] + A[k][j]`에서 오버플로가 발생할 수 있습니다. 현재는 `!= INT_MAX` 조건으로 잘 방어했지만, 더 안전한 `INF` 값을 별도로 정의하는 것이 좋습니다.
- 2) 인덱스/경계/오버플로/방문처리: 3중 루프 내 `if (i != j and j != k and k != i)` 조건은 불필요합니다. `i=j`, `k=i`, `k=j`인 경우는 최단 경로 갱신에 영향을 주지 않으므로, 이 조건들을 제거하면 코드가 더 간결하고 표준적인 형태가 됩니다.
- 3) 자료구조/성능 함정: 시작과 도착이 같은 노선이 여러 개 주어질 때 `min(A[a][b], c)` 처리는 필수적입니다. 이 부분은 현재 코드에서 잘 처리되었습니다.

## 즉시 수정 액션(최대 4개)
- 1. **`INF` 값 재정의:** `INT_MAX` 대신 `const int INF = 1e9 + 7;` 과 같이 오버플로에 더 안전한 상수를 정의하여 사용하세요.
- 2. **플로이드-워셜 로직 간소화:** 불필요한 조건을 제거하여 코드를 더 표준적이고 간결하게 수정하세요.
   ```cpp
   // 수정 전
   if (i != j and j != k and k != i and A[i][k] != INT_MAX and A[k][j] != INT_MAX)
       A[i][j] = min(A[i][j], A[i][k] + A[k][j]);

   // 수정 후 (INF 상수 사용)
   if (A[i][k] != INF && A[k][j] != INF)
       A[i][j] = min(A[i][j], A[i][k] + A[k][j]);
   ```
