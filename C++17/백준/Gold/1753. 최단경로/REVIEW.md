## 한 줄 결론
- (이 문제는) **다익스트라** / 복잡도: O(E log V) / 최단 경로 갱신이 없어도 `pq`에 `push`하는 실수가 가장 위험합니다.

## 더 빠른/더 안전한 풀이(필수)
- 현 접근(우선순위 큐를 이용한 다익스트라)이 거의 최선입니다.
- 이유: 음수 가중치가 없는 최단 경로 문제의 표준적인 O(E log V) 풀이입니다.

## 내가 지금 틀리기 쉬운 포인트 TOP3
- 1) 초기값/시작점/종료조건: `INF` 값을 `1e9`로 설정했는데, 간선 가중치 합이 `int` 범위를 넘지 않아 안전합니다.
- 2) 인덱스/경계/오버플로/방문처리: `visited` 배열로 이미 최단경로가 확정된 정점의 재방문을 막는 로직은 훌륭하고 효율적입니다.
- 3) 자료구조/성능 함정: **경로 갱신이 없는 경우에도 `pq.push`를 호출**하여 우선순위 큐에 중복/불필요한 원소가 쌓입니다. 이는 성능을 크게 저하시킬 수 있습니다.

## 즉시 수정 액션(최대 4개)
- `pq.push`는 반드시 `ans`(거리) 배열 값이 실제로 갱신되었을 때만 호출하도록 `if`문 안에 배치하세요.
- `min` 함수 대신 `if (new_dist < old_dist)` 형태의 명시적 조건문으로 코드를 변경하면 실수를 줄일 수 있습니다.
- 아래와 같이 수정하여 불필요한 `push` 연산을 방지하세요.
  ```cpp
  int u = a.second;
  for (edge b : A[u]) {
      int v = b.second;
      int weight = b.first;
      if (ans[v] > ans[u] + weight) {
          ans[v] = ans[u] + weight;
          pq.push({ans[v], v});
      }
  }
  ```
- 가독성을 위해 `ans`, `A` 같은 변수명 대신 `dist`, `adj` 등을 사용하는 것을 고려하세요.
