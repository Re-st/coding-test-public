- 현 접근이 거의 최선입니다. 우선순위 큐를 이용한 풀이는 로직이 정확하고 복잡도(O(N log N))도 최적입니다.
- 대안으로, 양수/음수/0/1을 별도 벡터에 나눠 담고 정렬하는 방식도 있습니다.
  - 아이디어: 양수는 내림차순, 음수는 오름차순 정렬 후 순서대로 2개씩 묶기
  - 복잡도: O(N log N)
  - 왜 이게 더 좋은지: 0과 남은 음수 처리 로직을 좀 더 명시적으로 구현해 실수를 줄일 수 있습니다.

## 내가 지금 틀리기 쉬운 포인트 TOP3
- 1) 0과 음수의 상호작용: 코드는 `0`을 음수용 min-heap(`neg`)에 함께 넣어 이 문제를 해결합니다. `0`은 음수 중 가장 큰 값이므로, 다른 음수들이 모두 짝지어진 후 마지막에 남아 홀수 개의 음수 중 가장 0에 가까운 수를 곱해 상쇄시키는 역할을 정확히 수행합니다.
- 2) 숫자 1의 특별 취급: `1`은 어떤 수와 곱하는 것보다 그냥 더하는 것이 항상 합을 크게 만듭니다 (`x*1` vs `x+1`). `else acc += 1;` 로직이 이 역할을 정확히 수행합니다.
- 3) 남은 원소 처리: 양수/음수 그룹의 개수가 홀수일 때, 마지막에 남는 1개는 곱할 대상이 없으므로 그대로 더해야 합니다. 코드의 `if (pos.size() == 1)` 와 `if (neg.size() == 1)` 부분이 이를 처리합니다.

# 제목: 정렬을 이용한 명시적 풀이
- 적용 조건: 0, 1, 양수, 음수 케이스를 명시적으로 분리해 가독성을 높이고 싶을 때.
- 핵심 아이디어: 양수(>1), 음수(<0)를 별도 벡터에 저장하고, 1과 0의 개수를 셉니다. 양수는 내림차순, 음수는 오름차순 정렬 후, 각 배열을 앞에서부터 2개씩 묶어 곱합니다. 홀수 개의 음수가 남아있고 0이 없을 때만 남은 음수를 더합니다.
- 복잡도: O(N log N)
- 구현 스케치(C++):
vector<int> pos, neg;
int one_cnt = 0, zero_cnt = 0;
long long ans = 0;
// 입력값을 pos(>1), neg(<0), one_cnt, zero_cnt 로 분리
// ...
sort(pos.rbegin(), pos.rend());
sort(neg.begin(), neg.end());

for (int i = 0; i < pos.size() / 2; ++i) ans += (long long)pos[2*i] * pos[2*i+1];
if (pos.size() % 2 != 0) ans += pos.back();

for (int i = 0; i < neg.size() / 2; ++i) ans += (long long)neg[2*i] * neg[2*i+1];
if (neg.size() % 2 != 0 && zero_cnt == 0) {
    ans += neg.back();
}
ans += one_cnt;
