## 내가 지금 틀리기 쉬운 포인트 TOP3
- 1) **동시성 문제**: 모든 기어의 회전 여부는 '회전 시작 전' 상태로 판단해야 합니다. 현재 재귀 코드는 일부 기어가 먼저 회전하고, 이 변경된 상태가 다른 기어의 판단에 영향을 줄 수 있는 치명적인 버그가 있습니다.
- 2) **재귀 호출 순서**: 재귀 호출 후 자신을 회전시켜, 전파의 끝(0, 3번)부터 회전이 실행됩니다. 이는 문제의 동시 회전 요구사항과 다릅니다.
- 3) **인덱스 혼동**: 톱니바퀴 번호(1~4)와 배열 인덱스(0~3), 맞닿는 톱니 인덱스(2, 6번)가 헷갈리기 쉬워 실수할 가능성이 있습니다. (현재 코드는 잘 처리했습니다)

## 즉시 수정 액션(최대 4개)
- 1. K번 루프 안에서, 회전 방향을 저장할 임시 배열 `int dirs[4] = {0};`을 선언하세요.
- 2. 명령 받은 톱니 `n`부터 시작해, 양옆으로 회전이 전파될지 **회전 전 상태**를 기준으로 `dirs` 배열에 기록합니다.
- 3. `dirs` 배열 작성이 끝나면, 별도 for문으로 `dirs[i]`가 0이 아닌 기어들을 실제 회전시키세요.
- 4. `move` 함수를 이 2단계 로직을 수행하는 비재귀 함수로 재작성하는 것을 권장합니다.

# 제목: 더 안전하고 명확한 2-Phase 풀이
- 적용 조건: 현재 문제처럼 상태 변화의 전파가 '동시에' 일어나야 할 때 항상 유리합니다.
- 핵심 아이디어: 회전 명령이 들어오면, (1) 어떤 기어들이 어떤 방향으로 돌지 먼저 전부 계산해서 배열에 저장하고, (2) 계산이 모두 끝난 뒤 그 배열을 보고 한꺼번에 실제 회전을 적용합니다.
- 복잡도: O(K)

void solve_one_turn() {
    int n, d;
    cin >> n >> d;
    --n; // 0-indexed

    // 1. 회전 방향 결정 단계
    int dirs[4] = {0};
    dirs[n] = d;

    // 왼쪽으로 전파
    for (int i = n; i > 0; --i) {
        if (t[i][6] != t[i - 1][2]) {
            dirs[i - 1] = -dirs[i];
        } else {
            break; // 전파 중단
        }
    }
    // 오른쪽으로 전파
    for (int i = n; i < 3; ++i) {
        if (t[i][2] != t[i + 1][6]) {
            dirs[i + 1] = -dirs[i];
        } else {
            break; // 전파 중단
        }
    }

    // 2. 실제 회전 적용 단계
    for (int i = 0; i < 4; ++i) {
        if (dirs[i] != 0) {
            rotate_gear(i, dirs[i]); // 기존 회전 로직 함수화
        }
    }
}
