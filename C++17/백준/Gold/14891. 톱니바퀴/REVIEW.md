## 더 빠른/더 안전한 풀이(필수)
- 아이디어: 회전 명령이 들어올 때마다, 어떤 톱니들이 어떤 방향으로 돌지 **먼저 모두 결정**한 뒤, **한 번에 회전**시키는 비재귀 방식이 더 안전합니다.
- 복잡도: O(K) (동일)
- 왜 이게 더 좋은지: 재귀의 복잡함과 상태 변경 타이밍 문제를 원천 차단하여 코드가 명확해지고 버그 가능성이 줄어듭니다.

## 내가 지금 틀리기 쉬운 포인트 TOP3
- 1) 전파 조건: 재귀 호출과 실제 회전 로직의 순서가 매우 중요합니다. 현재는 전파를 먼저 시키고 회전을 나중에 해서 맞지만, 순서가 바뀌면 회전된 상태를 기준으로 다음 전파를 결정하여 틀립니다.
- 2) 인덱스/경계: 3시 방향(인덱스 `2`)과 9시 방향(인덱스 `6`)을 혼동하기 쉽습니다. 또한, 문제의 1~4번 톱니를 0~3번 인덱스로 변환하는 과정(`n-1`)에서 실수가 나올 수 있습니다.
- 3) 회전 구현: 톱니바퀴 회전 시 임시 배열을 사용하지 않거나, `(i - d + 8) % 8` 같은 나머지 연산을 잘못하면 배열의 값이 덮어씌워져 틀립니다. 현재 구현은 안전합니다.

## 즉시 수정 액션(최대 4개)
- `move` 함수를 **“회전 방향 결정”**과 **“실제 회전”** 두 단계로 분리하는 것을 권장합니다. (위 '더 안전한 풀이' 참고)
- 매직 넘버 `2`, `6`을 각각 `RIGHT_POLE`, `LEFT_POLE` 같은 상수로 정의해 가독성을 높일 수 있습니다.
- C-style `#define` 대신 C++의 `const int`나 `enum class`를 사용하는 것이 타입 안정성 면에서 좋습니다.
- 회전 로직은 `std::deque`의 `push_front`/`pop_back` 등을 활용하면 더 간결하게 구현 가능합니다.

# 제목: 비재귀(iterative) 풀이 스케치
- 적용 조건: 재귀가 헷갈리거나, 상태 변경과 로직 결정을 명확히 분리하고 싶을 때 유리합니다.
- 핵심 아이디어: 1) 각 톱니의 회전 방향을 배열에 먼저 모두 계산. 2) 계산된 방향에 따라 모든 톱니를 한 번에 회전시킵니다.
- 복잡도: O(K) (기존과 동일)

void apply_rotation_command(int n, int d) {
    --n; // 0-indexed로 변환

    int dirs[4] = {0};
    dirs[n] = d;

    // 왼쪽으로 전파될 방향 결정
    for (int i = n - 1; i >= 0; --i) {
        if (t[i+1][6] != t[i][2]) dirs[i] = -dirs[i+1];
        else break;
    }

    // 오른쪽으로 전파될 방향 결정
    for (int i = n + 1; i < 4; ++i) {
        if (t[i-1][2] != t[i][6]) dirs[i] = -dirs[i-1];
        else break;
    }
    
    // 결정된 방향으로 실제 회전 적용
    for(int i = 0; i < 4; ++i) {
        if(dirs[i] != 0) rotate_gear(i, dirs[i]); // rotate_gear는 기존 회전 로직
    }
}
