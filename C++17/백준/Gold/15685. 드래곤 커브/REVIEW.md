## 한 줄 결론
- (이 문제는) **규칙 기반 시뮬레이션** / 복잡도: O(N * g * 2^g) / 비효율적인 점 집합 전체 회전

## 더 빠른/더 안전한 풀이(필수)
- **아이디어**: 점들의 좌표를 직접 회전하지 말고, 이동 방향의 순서를 규칙에 따라 생성한 뒤 경로를 한 번에 그립니다.
- **복잡도**: O(N * 2^g)
- **왜 이게 더 좋은지**: 복잡한 좌표 회전 계산과 `std::set`의 오버헤드가 없어 훨씬 빠르고 구현이 간결합니다.

## 내가 지금 틀리기 쉬운 포인트 TOP3
- 1) **초기값/시작점/종료조건**: 드래곤 커브들이 `visited` 배열을 공유하는 것은 문제 조건상 맞으며, 각 커브 정보는 루프마다 새로 입력받으므로 초기화 문제는 없습니다.
- 2) **인덱스/경계/오버플로/방문처리**: 사각형 개수를 세는 루프의 `i+1`, `j+1` 접근은 `visited[101][101]` 선언으로 안전합니다. 문제에서 "격자 밖으로 벗어나지 않는다"고 보장하므로 경로 생성 중 인덱스 오류 위험도 낮습니다.
- 3) **자료구조/성능 함정**: 현재 풀이는 세대가 증가할 때마다 점의 개수(`~2^g`)에 비례하는 연산을 반복합니다. 특히 `std::set`에 모든 점을 저장하고 회전시키는 것은 매우 비효율적입니다.

## 즉시 수정 액션(최대 4개)
- 1. 현재의 점 집합(`set<loc>`) 기반 접근 대신, 방향 리스트(`vector<int>`)를 생성하는 방식으로 전환하세요.
- 2. `g`세대 방향 리스트는 `g-1`세대 리스트 끝에, `g-1` 리스트를 뒤집고 각 방향을 `(d+1)%4` 변환한 리스트를 이어 붙여 만듭니다.
- 3. 최종 방향 리스트가 완성되면, 시작점 `(x, y)`에서 출발하여 경로를 한 번만 `visited` 배열에 표시합니다.
- 4. 제출 코드에 불필요한 주석(`// how to...`)은 제거하는 것이 좋습니다.

```REVIEW_EXAMPLE
# 제목: 방향 리스트 기반의 빠른 풀이 스케치
- 적용 조건: 현재 풀이보다 항상 유리합니다.
- 핵심 아이디어: 드래곤 커브의 생성 규칙은 "이전 세대 방향 리스트를 뒤집어 1씩 더한 방향을 뒤에 이어 붙이는 것"과 같습니다. 이 규칙으로 최종 방향 리스트를 O(2^g)에 구하고, 경로를 한 번에 그립니다.
- 복잡도: O(N * 2^g)
- 구현 스케치:
```cpp
// 각 드래곤 커브에 대해
vector<int> dirs;
dirs.push_back(d); // 0세대

for (int i = 0; i < g; ++i) {
    for (int j = dirs.size() - 1; j >= 0; --j) {
        dirs.push_back((dirs[j] + 1) % 4);
    }
}

// 방향 리스트를 따라 경로를 visited 배열에 표시
visited[x][y] = true;
for (int dir : dirs) {
    x += dx[dir];
    y += dy[dir];
    // 문제에서 격자 밖으로 벗어나지 않는다고 보장했지만, 방어적 코딩
    if (x >= 0 && x <= 100 && y >= 0 && y <= 100) {
        visited[x][y] = true;
    }
}
```
