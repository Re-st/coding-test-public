## 한 줄 결론
- **그리디/우선순위 큐** / 복잡도: O(N log N) / 누적 합 변수의 `int` 타입 오버플로

## 더 빠른/더 안전한 풀이(필수)
- 현 접근이 거의 최선입니다. 매 순간 가장 작은 두 묶음을 합치는 그리디 전략이 최적이며, 우선순위 큐는 이 전략을 O(N log N)에 가장 효율적으로 구현하는 자료구조입니다.

## 내가 지금 틀리기 쉬운 포인트 TOP3
- 1) 초기값/시작점/종료조건: N=1일 때 `is.size() > 1` 조건으로 루프를 건너뛰고 0을 출력하는 엣지 케이스는 잘 처리했습니다.
- 2) 인덱스/경계/오버플로/방문처리: **누적 비교 횟수 `acc`가 `int` 범위를 초과할 수 있습니다.** N 최댓값(10만)과 카드 크기(1000)를 고려하면 결과는 약 1.6 * 10^10까지 커질 수 있어 `long long`이 필수입니다.
- 3) 자료구조/성능 함정: Min-heap 구현을 위해 `Compare` 구조체를 사용한 것은 좋으나, 표준 라이브러리인 `greater<int>`를 쓰는 것이 더 간결합니다.

## 즉시 수정 액션(최대 4개)
- `int acc = 0;`를 `long long acc = 0;`으로 즉시 변경하세요.
- `priority_queue` 선언 시 `greater<int>`를 사용하면 `Compare` 구조체 없이 더 깔끔합니다.
  ```cpp
  // #include <functional> 필요
  priority_queue<int, vector<int>, greater<int>> pq;
  ```
- 전역 변수 `is`, `n`을 `main` 함수 내 지역 변수로 옮겨 코드의 명확성을 높이세요.
- 변수명 `is`를 `pq` 또는 `min_heap`처럼 의도가 명확한 이름으로 바꾸는 것을 권장합니다.
