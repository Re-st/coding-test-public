## 더 빠른/더 안전한 풀이(필수)
- 아이디어: 최소값은 DP로 구하는 것이 더 안전. `dp[i]` = 성냥 `i`개로 만들 수 있는 가장 작은 수. `dp[i]`는 `dp[i - cost[d]]`에 숫자 `d`를 붙이는 방식으로 갱신.
- 복잡도: O(N * L) (N: 성냥 수 ≤ 100, L: 최대 자릿수 ≤ 50)
- 왜 이게 더 좋은지: 복잡한 하드코딩 규칙 대신, 체계적이고 검증이 쉬운 방식으로 버그 가능성을 줄임.

## 내가 지금 틀리기 쉬운 포인트 TOP3
- 1) 최소값 로직: 현재 풀이의 `switch`문은 특정 `n` 값에 대한 패턴을 나열한 것으로 보입니다. 이 패턴이 모든 `n` (2~100)에 대해 최적임을 증명하기 어렵고, 엣지 케이스를 놓쳤을 수 있습니다.
- 2) 0 처리: 숫자는 0으로 시작할 수 없습니다. 최대값 생성은 간단하지만, 최소값 생성 시 `20`, `108`처럼 중간에 0을 효율적으로 사용하는 것이 핵심입니다. 현재 코드는 이 부분을 잘 처리하지만, 로직이 복잡해 실수가능성이 있습니다.
- 3) 자릿수와 값의 우선순위: 최소값을 찾을 땐 (1)자릿수가 적을수록, (2)자릿수가 같다면 사전 순으로 작을수록 좋습니다. `n=6`일 때 "6"(1자리)이 "111"(3자리)보다 작은 것이 그 예입니다.

```REVIEW_EXAMPLE
# 제목: 더 안전하고 명확한 DP 풀이 (최소값)
- 적용 조건: 현재 풀이의 복잡한 규칙을 이해하고 검증하기 어려울 때.
- 핵심 아이디어: `dp[i]`에 성냥 `i`개로 만들 수 있는 최소값을 문자열로 저장. `dp[i]`는 이전에 계산된 `dp[i-cost]`에 새 숫자를 붙여 만들 수 있는 모든 조합 중 가장 작은 값으로 갱신.
- 복잡도: O(N * L) (N: 성냥 수, L: 최대 자릿수)
- 구현 스케치(C++):
```cpp
#include <string>
#include <vector>
#include <algorithm>

// 비용: 0, 1, ..., 9
int cost[] = {6, 2, 5, 5, 4, 5, 6, 3, 7, 6};
string dp[101];

// string a가 b보다 작은지 비교 (자릿수 -> 사전순)
bool is_smaller(const string& a, const string& b) {
    if (a.length() != b.length()) return a.length() < b.length();
    return a < b;
}

void precompute_min_values() {
    for (int i = 0; i <= 100; ++i) dp[i] = string(51, '9');

    dp[2]="1"; dp[3]="7"; dp[4]="4"; dp[5]="2"; dp[6]="6"; dp[7]="8";
    
    for (int i = 8; i <= 100; ++i) {
        for (int d = 0; d <= 9; ++d) {
            if (i - cost[d] >= 2) {
                string candidate = dp[i - cost[d]] + to_string(d);
                if (is_smaller(candidate, dp[i])) {
                    dp[i] = candidate;
                }
            }
        }
    }
}
