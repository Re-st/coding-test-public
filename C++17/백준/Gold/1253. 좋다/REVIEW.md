## 한 줄 결론
- (이 문제는) **정렬 + 투 포인터** / 복잡도: O(N^2) / 제일 위험한 실수는 ‘자기 자신’을 합 계산에 포함시키는 것입니다.

## 더 빠른/더 안전한 풀이(필수)
- 현 접근이 거의 최선입니다. N ≤ 2,000 제약에서 O(N^2) 복잡도는 충분히 빠르며, 정렬 후 각 원소에 대해 투 포인터를 적용하는 것은 이 문제의 정석적인 풀이입니다.

## 내가 지금 틀리기 쉬운 포인트 TOP3
- 1) **자기 자신 제외 로직**: `if (a == i) a++; else if (b == i) b--;` 부분은 이 문제의 핵심입니다. 이 검사를 빼먹으면, `A[i]`를 만드는데 `A[i]` 자신이 사용될 수 있어 오답이 됩니다.
- 2) **포인터 진행 조건**: `while (a < b)` 조건이 중요합니다. `a <= b`가 되면, `a`와 `b`가 같은 인덱스를 가리켜 ‘서로 다른 두 수’라는 조건에 위배될 수 있습니다.
- 3) **중복 값 처리**: 문제의 "수의 위치가 다르면 값이 같아도 다른 수이다"라는 조건에 따라, 값이 같아도 모든 위치의 수를 검사해야 합니다. 현재 코드는 모든 `i`에 대해 독립적으로 검사하므로 이 조건을 잘 만족합니다.
