# 개선된 풀이 설명

## 현재 코드의 문제점

1. **불안전한 순열 처리**: `ans * 2`로 대칭성 가정 (단어 길이가 다르면 틀릴 수 있음)
2. **모호한 변수명**: `x3`, `x4`, `y3`, `y4` 등이 무엇을 의미하는지 불명확
3. **복잡한 인덱스 계산**: `w3[-y3]` 같은 음수 인덱스 사용

## 개선된 코드의 핵심

### 1. 명확한 변수명
```cpp
// 기존: x3, x4, y3, y4
// 개선: h1_v1 (H1에서 V1과 교차), h1_v2 (H1에서 V2와 교차)
```

### 2. 안전한 순열 처리
```cpp
// 기존: 일부만 처리하고 * 2
ans += inner(word[0], word[1], word[2], word[3]);
ans += inner(word[0], word[2], word[1], word[3]);
ans += inner(word[0], word[3], word[1], word[2]);
cout << ans * 2 << endl;

// 개선: 24가지 모두 처리
sort(words.begin(), words.end());
do {
    ans += countCrosswords(words[0], words[1], words[2], words[3]);
} while (next_permutation(words.begin(), words.end()));
```

### 3. 명확한 인덱스 계산
```cpp
// 기존: w3[-y3]
// 개선: 
int v1_idx_at_0 = -v1_start;  // 명시적으로 계산
if (V1[v1_idx_at_0] != H1[h1_v1]) continue;
```

## 로직 설명

크로스워드 구조:
```
H1:  [a][b][c][d]
      |     |
V1:   x     (없음)
      |
H2:  [e][f][g][h]
      |     |
V2:   (없음) y
```

1. **H1-V1 교차점** 고정: `h1_v1` 위치에서 V1이 H1과 만남
2. **V1 시작 위치** 결정: `v1_start` (음수 가능)
3. **H1-V2 교차점** 고정: `h1_v2` 위치에서 V2가 H1과 만남 (h1_v1과 최소 2칸 떨어져야 함)
4. **V2 시작 위치** 결정: `v2_start`
5. **H2-V1 교차점** 찾기: H2의 `h2_v1` 위치에서 V1과 만나는 y좌표 `cross_y`
6. **H2-V2 교차점** 계산: 평행선 성질로 `h2_v2 = h2_v1 + (h1_v2 - h1_v1)`

## 왜 더 나은가?

1. **안전성**: 모든 순열을 명시적으로 처리하여 버그 위험 감소
2. **가독성**: 변수명만 봐도 무엇을 의미하는지 알 수 있음
3. **유지보수**: 나중에 수정할 때 실수할 가능성 감소

## 복잡도

- **기존**: O(L^6) - 6중 루프
- **개선**: O(L^6) - 여전히 6중 루프지만 더 명확함

복잡도는 같지만, **코드 품질**이 크게 향상되었습니다!
