## 1) 요약
제출된 코드는 조합(Combination)을 이용해 N번째 감소하는 수의 자릿수를 먼저 결정하고, 이후 재귀적으로 각 자릿수를 계산하여 N번째 감소하는 수를 직접 구축하는 전략을 사용합니다. 예를 들어, k번째 두 자릿수 감소하는 수를 찾기 위해, 첫 자릿수가 무엇인지 조합을 이용해 범위를 좁혀나가는 방식입니다.

- **시간 복잡도:** O(1) (감소하는 수의 총개수가 1024개로 고정되어 있어, 입력 N에 상관없이 연산 횟수가 거의 일정합니다.)
- **공간 복잡도:** O(1) (재귀 호출 깊이가 최대 10으로 제한됩니다.)

## 2) 시간 / 공간 복잡도 점검
문제의 제약 조건 `N <= 1,000,000` 입니다. 감소하는 수의 총 개수는 10개의 숫자(0~9)에서 일부를 선택하여 만들 수 있는 수의 개수와 같습니다. 이는 각 숫자를 선택하거나 선택하지 않는 2가지 경우의 수의 조합이므로, 공집합(0으로 처리)을 포함하여 총 `2^10 = 1024`개입니다.

따라서 N의 최댓값은 1023이며, `N > 1023` 이면 감소하는 수가 존재하지 않습니다. 코드의 전체적인 연산 횟수는 N의 크기와 무관하게 감소하는 수의 자릿수(최대 10)와 사용되는 숫자의 개수(10개)에 의해 제한됩니다. 따라서 현재 접근 방식의 시간 및 공간 복잡도는 매우 안전합니다.

## 3) 구현/버그 리스크
현재 코드는 영리한 수학적 접근을 사용했지만, 몇 가지 심각한 버그와 위험 요소를 포함하고 있습니다.

### 가장 큰 위험: 정수 오버플로 (Integer Overflow)
- **문제점:** `get_num` 함수의 반환 타입과 중간 계산 과정이 `int`로 되어 있습니다. 가장 큰 감소하는 수인 `9876543210`은 `int`의 표현 범위(약 2.1 x 10^9)를 훨씬 초과합니다. 이로 인해 N이 큰 경우 오버플로가 발생하여 잘못된 결과가 출력됩니다.
- **수정 제안:** 감소하는 수를 저장하고 계산하는 모든 변수 및 함수 반환 타입을 `long long`으로 변경해야 합니다.
  ```cpp
  // 스니펫: 함수 시그니처 수정
  long long get_num(int n, int len, long long acc) {
      // ...
  }
  
  // main 함수 내에서도 long long 사용
  // cout << get_num(n, len, 0LL);
  ```

### 비효율적인 조합 계산
- **문제점:** `combination` 함수는 메모이제이션(memoization)이 없는 순수 재귀로 구현되어, `C(n, r)`을 계산할 때마다 동일한 부분 계산(e.g., `C(n-1, r-1)`)이 반복됩니다. 이 문제에서는 `n`이 10으로 작아 문제가 되지 않지만, 일반적인 코딩 테스트 환경에서는 시간 초과를 유발할 수 있는 비효율적인 구현입니다.
- **수정 제안:** 2차원 배열을 이용해 파스칼의 삼각형을 미리 계산해두는 것이 훨씬 효율적입니다.
  ```cpp
  // 스니펫: 조합 미리 계산
  long long C[11][11];
  void precompute_combinations() {
      for (int i = 0; i <= 10; ++i) {
          C[i][0] = 1;
          for (int j = 1; j <= i; ++j) {
              C[i][j] = C[i-1][j-1] + C[i-1][j];
          }
      }
  }
  // 이후 combination(n, r) 대신 C[n][r] 사용
  ```

### 복잡한 조합 논리
`combi_for_len`을 계산할 때 `combination(9, len) + combination(9, len - 1)`을 사용하셨습니다. 이는 파스칼의 항등식에 의해 `combination(10, len)`과 동일합니다. `len`개의 숫자를 10개(0~9) 중에서 고르는 경우의 수이므로, `combination(10, len)`으로 직접 계산하는 것이 더 직관적이고 가독성이 좋습니다.

### 가장 큰 수 처리 로직의 off-by-one 버그
- **문제점:** 코드는 N이 1022일 때 `9876543210`을 출력하고, N이 1023일 때는 `-1`을 출력합니다. 하지만 감소하는 수는 총 1024개 (0 포함, 0-indexed) 이므로, 가장 마지막 수의 인덱스는 1023입니다. 현재 로직은 이 마지막 케이스를 처리하지 못합니다.
- **원인:** `main` 함수의 `for` 루프가 끝났을 때 `n` 값으로 마지막 수를 판단하는 로직에 문제가 있습니다. `N=1022`가 입력되면 `n`이 `0`이 되고, `N=1023`이 입력되면 `n`이 `1`이 되어 `-1`이 출력됩니다.
- **해결 아이디어:** N이 1023보다 크면 초기에 `-1`을 출력하고 종료하는 것이 더 간단하고 안전합니다.

## 4) 엣지 케이스 체크리스트
- **최소 입력 (N=0):** 현재 코드에서 `if (n == 0)` 분기로 잘 처리되고 있습니다. **(통과)**
- **최대 입력 (N=1,000,000):** 감소하는 수는 1024개 뿐이므로, N이 1023을 초과하면 `-1`을 출력해야 합니다. 현재 코드는 이 경우 `-1`을 출력하지만, 그 과정이 다소 불안정합니다. (위 버그 리스크 참고)
- **경계값 (N=1022, N=1023):**
    - `N=1022` 입력 시, 코드는 `9876543210`을 출력합니다. 이는 인덱스가 1022인 수(9876543210 이전의 수)가 아니라 인덱스 1023인 수를 출력하는 것으로 보입니다. (오동작)
    - `N=1023` 입력 시, 코드는 `-1`을 출력합니다. 정답은 `9876543210`입니다. **(실패)**
- **자릿수가 바뀌는 경계 (N=9, N=10):**
    - `N=9`: 1자리 수의 마지막인 9를 잘 출력합니다.
    - `N=10`: 2자리 수의 처음인 10을 잘 출력합니다.

## 5) 리팩토링/가독성 개선 (선택)
현재의 조합론적 접근은 매우 복잡하여 버그를 찾고 수정하기 어렵습니다. 이 문제에 대한 더 간단하고 일반적인 해결책은 **BFS(너비 우선 탐색) 또는 백트래킹을 이용해 모든 감소하는 수를 생성**하는 것입니다.

1.  BFS 큐에 한 자릿수인 1부터 9까지 넣습니다.
2.  큐에서 숫자를 하나 꺼내 `result` 벡터에 추가합니다.
3.  꺼낸 수의 마지막 자릿수보다 작은 숫자들을 뒤에 붙여 새로운 수를 만들어 큐에 넣습니다. (예: `32`를 꺼냈다면, 마지막 자릿수 `2`보다 작은 `0`, `1`을 붙여 `320`, `321`을 만들어 큐에 넣습니다.)
4.  이 과정을 큐가 빌 때까지 반복합니다.
5.  `result` 벡터에 0을 추가하고 정렬하면 모든 감소하는 수를 오름차순으로 얻을 수 있습니다.

이 방법은 약 1024개의 수를 생성하고 정렬만 하면 되므로 매우 빠르며, 구현이 직관적이라 실수를 줄일 수 있습니다.

```cpp
// 스니펫: BFS를 이용한 생성 방식
#include <iostream>
#include <vector>
#include <queue>
#include <algorithm>

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int n;
    cin >> n;

    if (n > 1022) { // 0-indexed, 0~1022는 1023개. 9876543210이 1022번째. 아 0~9포함하면 1023개.
        cout << -1 << "\n";
        return 0;
    }

    vector<long long> decreasing_nums;
    queue<long long> q;

    for (int i = 0; i <= 9; ++i) {
        q.push(i);
        decreasing_nums.push_back(i);
    }

    while (!q.empty()) {
        long long current = q.front();
        q.pop();

        int last_digit = current % 10;

        for (int i = 0; i < last_digit; ++i) {
            long long next = current * 10 + i;
            decreasing_nums.push_back(next);
            q.push(next);
        }
    }

    sort(decreasing_nums.begin(), decreasing_nums.end());
    
    // N이 1023 이상이면 -1을 출력해야 하지만, 
    // 생성된 숫자의 개수가 N보다 작으면 -1을 출력하는 방식으로도 가능
    if (n >= decreasing_nums.size()) {
         cout << -1 << "\n";
    } else {
         cout << decreasing_nums[n] << "\n";
    }

    return 0;
}
```
*수정: 위 BFS 코드에서 `decreasing_nums`에 넣는 시점을 큐에 넣을 때로 통일하고, 0~9를 초기에 모두 넣는 것이 더 간단합니다. 정렬도 필요 없어집니다. BFS 순서가 곧 오름차순이기 때문입니다.*
