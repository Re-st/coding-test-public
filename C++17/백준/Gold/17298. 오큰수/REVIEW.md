## 내가 지금 틀리기 쉬운 포인트 TOP3
- 1) 초기값/시작점/종료조건: 정답 배열 `ans`를 -1로 미리 초기화하여, 스택에 끝까지 남아 오큰수가 없는 원소들을 올바르게 처리했습니다.
- 2) 인덱스/경계/오버플로/방문처리: 스택에 값만 저장하지 않고 `pair<인덱스, 값>`을 저장하여, 동일한 값을 가진 원소가 있어도 각자의 오큰수를 정확한 위치에 기록했습니다.
- 3) 자료구조/성능 함정: O(N^2)의 단순 탐색 대신 스택을 활용해 시간 복잡도를 O(N)으로 최적화했습니다. 각 원소는 스택에 최대 한 번씩만 PUSH/POP 됩니다.

## 즉시 수정 액션(최대 4개)
- 1. (스타일) `typedef` 대신 모던 C++ 스타일인 `using`을 사용하세요: `using Node = pair<int, int>;`
- 2. (성능) `push`와 생성자 호출 대신 `emplace`를 쓰면 더 간결하고 효율적입니다: `myStack.emplace(i, now);`
- 3. (가독성) 변수 `now`는 `for` 루프 안에서만 사용되므로, 루프 내부에 선언하여 변수의 유효 범위를 좁히는 것이 좋습니다.
- 4. (네이밍) `myStack`은 `stk` 또는 `s`와 같이 더 간결한 이름으로 바꿔도 좋습니다.
