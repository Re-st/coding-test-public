# AI 코드 리뷰

- Model: `models/gemini-2.5-pro`
- Source: `./C++17/백준/Bronze/1546. 평균/평균.cc`

## 1) 요약
제출된 코드는 N개의 점수를 배열에 모두 저장한 후, 두 번째 반복문에서 점수 총합과 최댓값을 찾아냅니다. 이후 `(총합 * 100 / 최댓값) / 과목 수` 라는 수식을 이용해 조작된 점수의 평균을 계산하는 전략을 사용합니다.

- 시간 복잡도: O(N), 공간 복잡도: O(N)

## 2) 시간 / 공간 복잡도 점검
문제의 제약 조건은 `N <= 1000` 입니다.

- **시간 복잡도**: O(N)은 약 1000번의 연산에 해당하므로 시간 제한 내에 충분히 동작합니다.
- **공간 복잡도**: O(N)은 `int` 배열 기준 약 4KB (1001 * 4 bytes)의 메모리를 사용하며, 이 역시 메모리 제한에 전혀 문제 되지 않습니다.

하지만, 모든 점수를 저장할 필요 없이 입력을 받으면서 총합과 최댓값을 동시에 계산하면 공간 복잡도를 O(1)로 최적화할 수 있습니다. 코딩 테스트에서는 이처럼 **불필요한 메모리 사용을 줄이는 습관**을 들이는 것이 좋습니다.

## 3) 구현/버그 리스크
현재 코드의 로직은 정확하며 특별한 버그 위험은 보이지 않습니다. 특히 칭찬할 점은 다음과 같습니다.

- **실수형 연산**: `sum`과 `m`을 `double`로 선언하고, `100.0`을 곱하여 전체 수식을 실수 연산으로 처리한 덕분에 정수 나눗셈으로 인한 오차를 성공적으로 방지했습니다.

다만, 개선할 수 있는 부분은 있습니다.
- **불필요한 배열 사용**: 점수를 읽는 루프와, 총합/최댓값을 구하는 루프가 분리되어 있습니다. 이로 인해 `num` 배열에 모든 점수를 저장하게 되는데, 사실 이 배열은 필요하지 않습니다.

```cpp
// 현재 코드
for (int i = 0; i < n; i++) cin >> num[i];
for (int i = 0; i < n; i++) {
    // ... sum, m 계산 ...
}

// 개선 제안 (O(1) 공간 복잡도)
// 배열 num[1001] 없이 바로 처리
double score;
for (int i = 0; i < n; i++) {
    cin >> score;
    if (score > m) m = score;
    sum += score;
}
```

## 4) 엣지 케이스 체크리스트
- **최소 입력**: N=1일 때, `sum`과 `m`은 유일한 점수 값이 되고, `sum / m`은 1이 되어 평균 100이 정상적으로 출력됩니다.
- **최대 입력**: N=1000이고 모든 점수가 100일 때, `sum`은 100,000이 되며 `int`나 `double` 범위 내에 있어 오버플로 문제가 없습니다.
- **0점 포함**: 문제에서 "적어도 하나의 값은 0보다 크다"고 명시하여 최댓값 `m`이 0이 되는 경우를 배제했습니다. 따라서 0으로 나누는 (Division by Zero) 위험은 없습니다.
- **중복 값**: 모든 점수가 동일할 경우에도 `m`과 `sum`이 올바르게 계산되어 정상 동작합니다.

전반적으로 엣지 케이스에 대해 안전하게 구현되었습니다.

## 5) 리팩토링/가독성 개선 (선택)
코드가 간결하여 큰 리팩토링은 필요 없지만, 위 3번 항목에서 제안한 것처럼 두 개의 반복문을 하나로 합치는 것이 가장 효과적인 개선입니다.

- **변수 `tmp` 제거**: `tmp` 변수 없이 `num[i]`를 직접 사용하면 코드가 한 줄 더 간결해집니다.
  ```cpp
  // 현재 코드
  tmp = num[i];
  if (tmp > m) m = tmp;
  sum += tmp;

  // 개선 제안
  if (num[i] > m) m = num[i];
  sum += num[i];
  ```

## 6) 한 줄 평
조작된 평균을 구하는 수식을 정확히 도출하고 실수 연산까지 잘 처리했지만, 불필요한 배열 사용으로 O(1) 공간 복잡도로 풀 수 있는 기회를 놓친 점이 아쉽습니다.
