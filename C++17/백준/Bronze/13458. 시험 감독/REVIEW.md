## 1) 요약
제출된 코드는 각 시험장마다 필요한 감독관 수를 계산하는 전략을 사용합니다. 먼저 모든 시험장에 총감독관 1명씩을 배정하고(`ans = n`으로 초기화), 총감독관이 감당하고 남은 응시자 수(`pop - b`)에 대해 필요한 부감독관 수를 계산하여 누적합니다. 부감독관 수는 올림 나눗셈 `(남은 인원 + C - 1) / C` 공식을 이용해 효율적으로 구했습니다.

- 시간 복잡도: O(N), 공간 복잡도: O(N)

## 2) 시간 / 공간 복잡도 점검
문제의 제약 조건은 `N`이 최대 1,000,000입니다.

- **시간 복잡도**: 코드는 N개의 시험장 정보를 한 번 순회하며 계산하므로 시간 복잡도는 O(N)입니다. N=1,000,000일 때 약 10^6번의 연산으로 시간 제한 내에 충분히 통과할 수 있습니다.
- **공간 복잡도**: 응시자 수를 저장하기 위해 `vector<int> a`를 사용하므로 O(N)의 공간을 사용합니다. `sizeof(int) * 1,000,000`는 약 4MB로 메모리 제한에도 안전합니다.

복잡도 측면에서 매우 효율적인 풀이입니다.

## 3) 구현/버그 리스크
- **(Good) 오버플로 방지**: 감독관의 총 수를 저장하는 `ans` 변수를 `long long`으로 선언한 점이 매우 훌륭합니다. 최악의 경우(N=10^6, A=10^6, B=1, C=1) `ans`의 값은 약 `10^6 * 10^6 = 10^12`에 육박하여 `int` 범위를 초과할 수 있습니다. `long long`을 사용하여 오버플로를 사전에 방지했습니다.

- **(Good) 음수 처리**: 총감독관만으로 모든 응시자를 감시할 수 있는 경우(`pop - b <= 0`), `pop - b`는 음수가 될 수 있습니다. 이때 `max(0, pop - b)`를 사용하여 남은 인원을 0으로 처리함으로써, 부감독관 수가 음수가 되는 버그를 효과적으로 막았습니다.

- **불필요한 코드**: `static char d[10001];` 배열은 코드 내에서 전혀 사용되지 않습니다. 의도를 알 수 없으며, 혼란을 줄 수 있으므로 제거하는 것이 좋습니다.

## 4) 엣지 케이스 체크리스트
현재 풀이는 주요 엣지 케이스들을 잘 처리하고 있습니다.
- **최소/최대 입력**: N=1, N=1,000,000 모두 잘 동작합니다.
- **총감독관만으로 충분한 경우**: `A_i <= B`인 경우 `max(0, pop - b)`가 0이 되어 부감독관이 추가되지 않습니다. (정상)
- **부감독관이 필요한 경우 (나누어 떨어짐)**: `A_i = 10, B = 4, C = 3`일 때, 남은 인원 6명에 부감독관 2명이 필요합니다. `(6 + 3 - 1) / 3 = 2`로 정확히 계산됩니다. (정상)
- **부감독관이 필요한 경우 (나머지 발생)**: `A_i = 10, B = 4, C = 4`일 때, 남은 인원 6명에 부감독관 2명이 필요합니다. `(6 + 4 - 1) / 4 = 2`로 정확히 계산됩니다. (정상)

전반적으로 로직이 견고하여 특별히 문제 될 만한 엣지 케이스는 보이지 않습니다.

## 5) 리팩토링/가독성 개선 (선택)
- **입력 방식**: `while (n--) cin >> a[n];` 구문은 동작은 하지만, `n` 변수를 입력받은 크기 정보와 루프 카운터로 동시에 사용하여 다소 비직관적입니다. 일반적인 `for` 루프를 사용하면 가독성을 더 높일 수 있습니다.

  ```cpp
  // 제안
  int num_locations;
  cin >> num_locations;
  vector<int> a(num_locations);
  long long ans = num_locations; // 총감독관은 시험장 수만큼 필요

  for (int i = 0; i < num_locations; ++i) {
      cin >> a[i];
  }

  // 이후 로직 동일
  ```

- **불필요한 배열 제거**: 위에서 언급했듯이, 사용하지 않는 `d` 배열은 삭제하는 것이 좋습니다.
