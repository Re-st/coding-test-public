## 1) 요약
N개의 정수를 입력받아 `std::vector`에 저장한 뒤, 중첩 for문을 이용한 O(N^2) 시간 복잡도의 정렬 알고리즘을 직접 구현하여 오름차순으로 정렬 후 출력하는 코드입니다.

- **시간 복잡도**: O(N^2)
- **공간 복잡도**: O(N)

## 2) 시간 / 공간 복잡도 점검
문제에서 N의 최대값은 1,000으로 주어졌습니다.

- **시간 복잡도**: O(N^2) 알고리즘은 최악의 경우 약 1,000 * 1,000 = 1,000,000번의 연산을 수행합니다. 이는 일반적인 코딩 테스트의 시간제한(1초에 약 1억 번 연산) 내에 충분히 통과할 수 있는 수준입니다. 따라서 **시간 복잡도는 안전합니다.**
- **공간 복잡도**: N개의 정수를 저장하기 위해 `std::vector`를 사용하므로 O(N)의 공간을 사용합니다. N=1,000일 때 약 4KB 정도의 메모리를 사용하므로 **공간 복잡도도 안전합니다.**

## 3) 구현/버그 리스크
현재 코드에서 몇 가지 잠재적인 리스크와 오해를 부를 수 있는 패턴이 보입니다.

**1. 입력 방식**
```cpp
while (n--) cin >> myvec[n];
```
이 코드는 `n`의 값을 감소시키면서 벡터의 뒤쪽부터 값을 채워 넣습니다. (예: `n=5`일 때 `myvec[4]`, `myvec[3]`, ..., `myvec[0]` 순으로 입력)
- **가독성 저하**: 일반적인 `for (int i=0; i<n; ++i)` 구문보다 코드의 의도를 파악하기 어렵습니다.
- **원본 `n` 값 소실**: `while` 루프가 끝나면 `n`의 값은 0이 됩니다. 만약 코드 뒷부분에서 원래 수의 개수 `n`이 필요했다면 버그의 원인이 될 수 있습니다. (이 코드에서는 다행히 `myvec.size()`를 사용해서 문제가 없습니다.)

**2. 비표준 정렬 알고리즘**
```cpp
for (int i = 1; i < myvec.size(); i++) {
    for (int j = 0; j < i; j++) {
        if (myvec[i] < myvec[j]) {
            int tmp = myvec[j];
            myvec[j] = myvec[i];
            myvec[i] = tmp;
        }
    }
}
```
이 정렬 로직은 동작은 하지만, 일반적인 삽입 정렬이나 버블 정렬과는 다른 독특한 형태입니다.
- **로직의 복잡성**: `i`번째 원소를 `0`부터 `i-1`까지의 원소와 비교하며, 조건이 맞을 때마다 계속 `swap`합니다. 이 과정에서 `myvec[i]`의 값이 계속 바뀌므로 알고리즘의 동작을 예측하고 정확성을 증명하기가 어렵습니다.
- **비효율**: 예를 들어 `[5, 4, 3, 2, 1]`을 정렬할 때, `i=4` (값 1)일 때, 내부 루프에서 불필요한 `swap`이 여러 번 발생합니다. 표준 삽입 정렬은 `i`번째 원소를 올바른 위치에 "한 번만" 삽입하는 것과 대조적입니다.

이러한 비표준 알고리즘은 의도치 않은 버그를 만들 가능성이 있으며, 다른 사람이 코드를 이해하기 어렵게 만듭니다.

## 4) 엣지 케이스 체크리스트
- **최소 입력 (N=1)**: `for`문이 실행되지 않고 바로 출력되므로 정상 동작합니다.
- **최대 입력 (N=1000)**: 시간 복잡도 분석에서 확인했듯이 안전합니다.
- **이미 정렬된 입력**: `if (myvec[i] < myvec[j])` 조건이 항상 거짓이므로 `swap`은 일어나지 않지만, 불필요한 비교 연산은 O(N^2)만큼 수행됩니다.
- **역순으로 정렬된 입력**: `swap`이 매우 빈번하게 발생하며 최악의 성능을 보입니다.
- **음수/0 포함**: `int` 타입과 `<` 비교 연산자는 음수와 0을 올바르게 처리하므로 문제없습니다.

제출된 코드는 위 엣지 케이스들에서 모두 정답을 출력할 것으로 보입니다.

## 5) 리팩토링/가독성 개선
코드를 더 안정적이고, 간결하며, 효율적으로 만들 수 있는 몇 가지 제안입니다.

**1. 표준 정렬 함수 `std::sort` 사용**
C++ 표준 라이브러리에서 제공하는 `std::sort`를 사용하는 것이 가장 좋은 방법입니다.
- **성능**: 평균적으로 O(N log N)의 시간 복잡도를 보장하여 O(N^2)보다 훨씬 빠릅니다.
- **안정성**: 수많은 테스트를 거쳐 검증된 함수이므로 직접 구현한 로직보다 버그 가능성이 거의 없습니다.
- **가독성**: `sort(vec.begin(), vec.end())` 한 줄로 "정렬한다"는 의도를 명확하게 보여줍니다.

```cpp
// <algorithm> 헤더 필요
#include <iostream>
#include <vector>
#include <algorithm> // std::sort를 위해 추가

using namespace std;

int main() {
    ios::sync_with_stdio(false); cin.tie(NULL); cout.tie(NULL);
    
    int n;
    cin >> n;
    
    vector<int> numbers(n);
    for (int i = 0; i < n; ++i) {
        cin >> numbers[i];
    }
    
    // std::sort 사용
    sort(numbers.begin(), numbers.end());
    
    for (int i = 0; i < numbers.size(); ++i) {
        cout << numbers[i] << '\n';
    }
    
    return 0;
}
```
**2. 입력 루프 개선**
앞서 언급했듯, 보다 명시적이고 안전한 `for` 루프를 사용하는 것이 좋습니다.

```cpp
// 기존 코드
// while (n--) cin >> myvec[n];

// 개선 제안
vector<int> numbers(n); // 변수명도 myvec -> numbers로 개선
for (int i = 0; i < n; ++i) {
    cin >> numbers[i];
}
```

## 6) 한 줄 평
"O(N^2) 정렬을 직접 구현하여 문제를 해결한 점은 좋았지만, 가독성과 성능, 안정성을 위해 표준 라이브러리의 `std::sort`를 활용하는 것을 적극 권장합니다."
