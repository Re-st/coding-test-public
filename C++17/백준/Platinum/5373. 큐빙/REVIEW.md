- 2) 인덱스/경계/오버플로: `outLocs` 맵에 288개(6면*12조각*2좌표)의 인덱스를 직접 입력하여, 단 하나의 오타만 있어도 오답으로 이어집니다. 예를 들어 `L` 회전 시 `B`면의 `(3, 8), (3, 5), (3, 2)` 순서가 맞는지 검증이 사실상 불가능합니다.
- 3) 자료구조/성능 함정: `map`을 사용했지만 키 개수가 적어 성능 문제는 없습니다. 위험은 자료구조의 내용(하드코딩된 좌표) 자체의 정확성에 있습니다.

# 제목: 하드코딩을 제거한 규칙 기반 풀이
- 적용 조건: 현재 풀이처럼 모든 좌표를 하드코딩하는 것이 부담스럽거나 디버깅이 어려울 때 유리합니다.
- 핵심 아이디어: 큐브를 `char cube[6][3][3]`으로 관리하고, 회전 함수는 (1) 해당 면의 3x3 배열 자체를 90도 회전, (2) 인접한 4개 면의 특정 행/열을 임시 배열에 저장 후, 순서에 맞게 복사하는 두 단계로 구성합니다.
- 복잡도: O(T*N)
- 구현 스케치(C++):
```cpp
// 0:U, 1:D, 2:F, 3:B, 4:L, 5:R
char cube[6][3][3];

// 3x3 면을 시계방향으로 90도 회전
void rotate_face_cw(int face_idx) {
    char tmp[3][3];
    for (int i = 0; i < 3; ++i)
        for (int j = 0; j < 3; ++j)
            tmp[i][j] = cube[face_idx][i][j];

    for (int i = 0; i < 3; ++i)
        for (int j = 0; j < 3; ++j)
            cube[face_idx][i][j] = tmp[2 - j][i];
}

void rotate_U_plus() {
    rotate_face_cw(0); // U면 자체 회전

    char tmp_row[3];
    for (int i = 0; i < 3; ++i) tmp_row[i] = cube[4][0][i]; // L면 윗줄 임시 저장

    // B(3)->L(4), R(5)->B(3), F(2)->R(5)
    for (int i = 0; i < 3; ++i) cube[4][0][i] = cube[3][0][i];
    for (int i = 0; i < 3; ++i) cube[3][0][i] = cube[5][0][i];
    for (int i = 0; i < 3; ++i) cube[5][0][i] = cube[2][0][i];
    
    for (int i = 0; i < 3; ++i) cube[2][0][i] = tmp_row[i]; // 임시(L) -> F(2)
}
```
