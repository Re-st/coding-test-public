## 한 줄 결론
- (이 문제는) **Two-Pointers** / 복잡도: O(N) / `status` 변수와 중첩 `while` 로직이 비직관적이라 실수 위험이 있습니다.

## 더 빠른/더 안전한 풀이(필수)
- **아이디어**: N을 연속된 자연수의 합으로 표현하는 방법의 수는 N의 홀수 약수의 개수와 같습니다.
- **복잡도**: O(sqrt(N))
- **왜 이게 더 좋은지**: N=10^7일 때 O(N)은 약 10^7번 연산이지만, O(sqrt(N))은 약 3,200번 연산으로 훨씬 빠릅니다.

## 내가 지금 틀리기 쉬운 포인트 TOP3
1. **로직 복잡성**: `status` 변수를 써서 합계와 목표값의 차를 관리하는 방식은 일반적이지 않아 의도를 파악하고 디버깅하기 어렵습니다.
2. **증감 연산자**: `status += ++e`, `status -= s++` 같이 한 줄에 증감 연산과 다른 연산을 섞어 쓰면 실행 순서가 헷갈려 버그를 유발하기 쉽습니다.
3. **성능 함정**: 현재 코드는 O(N)으로 N=10^7에서 아슬아슬하게 통과하지만, N 범위가 더 컸다면 시간 초과가 났을 문제입니다. 더 효율적인 수학적 풀이의 가능성을 먼저 탐색하는 것이 좋습니다.

## 즉시 수정 액션(최대 4개)
1. 불필요한 `<vector>` 헤더를 제거하세요.
2. 주석 처리된 코드(`/*if (!(n % 3)) ...*/`)는 혼란을 주므로 삭제하세요.
3. (가독성) 아래와 같이 `sum` 변수를 직접 사용하고 `if-else` 구조로 바꾸면 코드가 훨씬 명확해집니다.
    ```cpp
    int s = 1, e = 1, sum = 1, acc = 0;
    while (s <= n) {
        if (sum == n) { acc++; sum -= s++; }
        else if (sum < n) { if(e < n) sum += ++e; else break; }
        else { sum -= s++; }
    }
    ```

# 제목: 더 빠른 풀이 스케치 (수학적 접근)
- 적용 조건: N의 크기가 커서 O(N) 풀이가 시간 초과될 수 있거나, 더 빠른 실행 시간이 필요할 때.
- 핵심 아이디어: N을 연속된 자연수의 합으로 표현하는 방법의 수는 N의 홀수 약수의 개수와 같다는 수학적 원리를 이용합니다.
- 복잡도: O(sqrt(N))
- 구현 스케치:
```cpp
#include <iostream>

int main() {
    std::ios_base::sync_with_stdio(false);
    std::cin.tie(NULL);
    int n;
    std::cin >> n;

    int count = 0;
    for (int k = 1; k * (k + 1) / 2 <= n; ++k) {
        if ((n - k * (k - 1) / 2) % k == 0) {
            count++;
        }
    }
    std::cout << count << "\n";
    return 0;
}
```
