## 한 줄 결론
- (이 문제는) **다이나믹 프로그래밍(피보나치)** / 복잡도: O(N) / N이 90일 때 `long long`을 쓰지 않으면 오버플로가 나는 것이 제일 위험한 실수입니다.

## 더 빠른/더 안전한 풀이(필수)
- 현 접근이 거의 최선입니다. N이 90으로 작아 O(N) DP가 충분히 빠르고, 2차원 DP 상태(`dp[i][끝자리]`)를 1차원으로 최적화하여 공간/구현도 효율적입니다.

## 내가 지금 틀리기 쉬운 포인트 TOP3
- 1) 자료형/오버플로: N이 90일 때 결과값은 약 2.88 x 10^18으로 `long long`의 최대치(약 9 x 10^18)에 근접합니다. `int`나 `long`을 사용하면 100% 오버플로가 발생합니다.
- 2) 초기값/점화식: 점화식은 `dp[i] = dp[i-1] + dp[i-2]`가 맞지만, 실제 문제의 초기값은 `dp[1]=1`, `dp[2]=1`입니다. 코드의 `ans[0]=0, ans[1]=1`은 이 수열을 정확히 생성하지만, `ans[0]`는 실제 문제와 무관한 계산용 값임을 인지해야 합니다.
- 3) DP 상태 정의: 이 문제는 끝자리가 0인지 1인지에 따라 점화식이 갈리는 2차원 DP가 더 정석적인 출발점입니다. `dp[i] = dp[i-1] + dp[i-2]`는 이 2차원 DP를 1차원으로 최적화한 결과로, 바로 도출하려다 논리적 실수를 할 수 있습니다.

## 즉시 수정 액션(최대 4개)
- `#define MAX_INT 1000000001` 코드를 삭제하세요 (사용되지 않는 매크로).
- `ans` 변수명을 `dp`로 바꾸면 DP 문제임이 더 명확해집니다.
- N의 최대값이 90으로 작고 고정되어 있으므로, `vector` 대신 `long long dp[91];` 같은 정적 배열이 더 간결하고 빠릅니다.
- `ans[0]`의 초기값을 설정하는 대신, 아래처럼 N=1 예외처리를 하고 루프를 2부터 시작하면 점화식의 의미가 더 직관적으로 드러납니다.
```cpp
// 스니펫 제안
long long dp[91];
dp[1] = 1;
for (int i = 2; i <= n; i++) {
    dp[i] = dp[i-1] + dp[i-2]; // dp[0]이 0으로 자동 초기화되어 잘 동작
}
cout << dp[n];
```
