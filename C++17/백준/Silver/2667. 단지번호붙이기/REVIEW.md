  - 왜 이게 더 좋은지: 역할 분리가 명확해져 코드 이해가 쉽고, 잠재적인 버그(e.g., 시작점 카운트 누락)가 줄어듭니다.

# 제목: 더 안전하고 직관적인 DFS 풀이
- 적용 조건: 그래프 탐색 로직을 한 곳으로 모아 코드의 응집도를 높이고 싶을 때.
- 핵심 아이디어: `main`은 탐색 시작점만 찾고, 실제 탐색과 단지 내 집의 수 계산은 `dfs` 함수가 온전히 책임지도록 역할을 분리합니다. `dfs` 함수는 방문한 집의 수를 반환합니다.
- 복잡도: O(N²)
- 구현 스케치(C++):
```cpp
int n;
string board[25];
bool visited[25][25];
int dx[] = {0, 0, 1, -1};
int dy[] = {1, -1, 0, 0};

int dfs(int x, int y) {
    if (x < 0 || x >= n || y < 0 || y >= n || visited[x][y] || board[x][y] == '0') {
        return 0;
    }
    visited[x][y] = true;
    int count = 1;
    for (int i = 0; i < 4; ++i) {
        count += dfs(x + dx[i], y + dy[i]);
    }
    return count;
}

int main() {
    // ... 입력 처리
    vector<int> danji_sizes;
    for (int i = 0; i < n; ++i) {
        for (int j = 0; j < n; ++j) {
            if (board[i][j] == '1' && !visited[i][j]) {
                danji_sizes.push_back(dfs(i, j));
            }
        }
    }
    // ... 정렬 및 출력 처리
}
