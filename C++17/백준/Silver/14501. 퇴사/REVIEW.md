## 한 줄 결론
- (이 문제는) **DP(동적 계획법)** / 복잡도: O(N^2) / DP 점화식의 논리적 오류가 가장 위험합니다.

## 더 빠른/더 안전한 풀이(필수)
- **아이디어**: `dp[i]`를 "i일부터 마지막 날까지 얻을 수 있는 최대 이익"으로 정의하고, 뒤에서부터(`i=N`부터 1까지) 계산합니다.
- **복잡도**: O(N)
- **왜 이게 더 좋은지**: O(N)으로 더 빠르며, 'i일에 상담을 한다/안한다'의 선택 구조가 명확해져 코드가 단순해지고 버그 가능성이 줄어듭니다.

## 내가 지금 틀리기 쉬운 포인트 TOP3
- 1) **DP 점화식 오류**: `maxp[i]`를 계산하는데 `p[i]`가 아닌 과거의 `p[j]`를 사용하는 로직은 문제입니다. `i`번째 날의 상담을 포함할지 여부를 결정하는 구조가 아니므로 최대 이익을 보장할 수 없습니다.
- 2) **경계 조건**: `i`번째 상담을 선택하면 다음 상담은 `i + t[i]`일부터 가능합니다. 이 날짜가 `n`을 초과하는 경우에 대한 처리가 현재 로직에서는 불분명합니다.
- 3) **결과 집계**: 마지막에 `maxp` 배열 전체를 순회하며 최댓값을 찾는 방식은 DP 상태 정의가 불명확하기 때문에 필요해진 과정입니다. 올바른 DP는 보통 `dp[0]` 또는 `dp[1]`에서 바로 답을 얻습니다.

## 즉시 수정 액션(최대 4개)
- **DP 정의 변경**: `dp[i]`를 "i일 부터 퇴사일까지 얻을 수 있는 최대 수익"으로 재정의하세요.
- **반복문 방향 변경**: `for(int i = n-1; i >= 0; i--)`와 같이 역방향으로 순회하며 DP 테이블을 채우세요.
- **점화식 수정**: `i`번째 상담을 할 수 없다면(`i + t[i] > n`) `dp[i] = dp[i+1]`이 되고, 할 수 있다면 `dp[i] = max(dp[i+1], p[i] + dp[i + t[i]])`가 됩니다.
- **초기화**: `dp` 배열을 `n+1` 또는 `n+10` 등 충분한 크기로 잡고 0으로 초기화하세요. `dp[n]`은 0입니다.

```REVIEW_EXAMPLE
# 제목: 더 효율적이고 직관적인 DP 풀이 (역방향)
- 적용 조건: N 값에 상관없이 항상 더 직관적이고 효율적입니다.
- 핵심 아이디어: `dp[i]`를 "i일부터 마지막 날까지 얻을 수 있는 최대 이익"으로 정의합니다. `i`를 `N`부터 1까지 거꾸로 순회하며, `i`일의 상담을 (1) 하거나 (2) 안 하거나 두 경우 중 최댓값을 `dp[i]`에 저장합니다.
- 복잡도: O(N)
- 구현 스케치(C++):
```cpp
#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int n;
    cin >> n;

    vector<int> t(n), p(n);
    for (int i = 0; i < n; i++) {
        cin >> t[i] >> p[i];
    }

    vector<int> dp(n + 1, 0); // dp[i]: i일부터 얻을 수 있는 최대 수익

    for (int i = n - 1; i >= 0; i--) {
        // i번째 날에 상담을 시작해서 퇴사일(n) 안에 끝낼 수 있는 경우
        if (i + t[i] <= n) {
            // i번째 상담을 하는 경우 vs 하지 않는 경우
            dp[i] = max(p[i] + dp[i + t[i]], dp[i + 1]);
        } else {
            // i번째 상담을 하면 퇴사일을 넘기므로 할 수 없음
            dp[i] = dp[i + 1];
        }
    }

    cout << dp[0] << endl;

    return 0;
}
