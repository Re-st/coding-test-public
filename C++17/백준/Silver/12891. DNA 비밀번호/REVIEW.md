## 한 줄 결론
- **슬라이딩 윈도우** / 복잡도: O(S) / 윈도우 초기화 및 이동 로직의 off-by-one 실수

## 더 빠른/더 안전한 풀이(필수)
- 현 접근(O(S) 슬라이딩 윈도우)이 거의 최선입니다. 모든 문자를 한 번씩은 확인해야 하므로 O(S)가 이론적 하한선입니다.

## 내가 지금 틀리기 쉬운 포인트 TOP3
- 1) 초기값/시작점/종료조건: 첫 `p-1`개만 먼저 처리하는 방식은 일반적이지 않아, 루프 시작/종료 조건에서 실수가 발생하기 쉽습니다.
- 2) 변수명 혼동: 입력 변수 `s`, `p`가 문제 설명의 DNA 문자열 길이 `|S|`, 부분문자열 길이 `|P|`와 반대로 사용되어 코드 해석에 혼란을 줍니다.
- 3) 자료구조/성능 함정: `dnaint` 벡터는 불필요하게 O(S)의 추가 메모리를 사용합니다. DNA 문자는 즉시 인덱스로 변환 가능합니다.

## 즉시 수정 액션(최대 4개)
- 변수명을 `S_len`, `P_len` 등으로 명확히 하여 문제의 기호와 일치시키세요.
- `dnaint` 벡터를 제거하고, 루프 내에서 문자를 직접 인덱스로 변환하여 메모리 사용량을 O(1)으로 최적화하세요.
- 윈도우 처리 로직을 **(1) 첫 윈도우(0 ~ P-1) 구성 및 검사**, **(2) 이후 슬라이딩(P ~ S-1)**으로 명확히 분리해 가독성을 높이세요.
- 반복되는 유효성 검사 로직(`if (acgt[0] >= ...)`)을 별도의 `check()` 함수로 분리하는 것을 고려하세요.

# 제목: 더 단순하고 안전한 슬라이딩 윈도우 구현
- 적용 조건: 현재 풀이처럼 슬라이딩 윈도우를 사용할 때, 코드 구조를 단순화하여 실수를 줄이고 싶을 경우.
- 핵심 아이디어: 첫 윈도우(크기 P)를 먼저 완전히 처리하고, 그 후 한 칸씩 이동하며 추가/제거되는 원소만 갱신합니다.
- 복잡도: O(S)
- 구현 스케치(C++):
  ```cpp
  // char to index helper
  int char_to_idx(char c) {
      if (c == 'A') return 0;
      if (c == 'C') return 1;
      if (c == 'G') return 2;
      return 3; // 'T'
  }
  
  // 1. 첫 윈도우 (0 ~ P-1) 처리
  for (int i = 0; i < P_len; ++i) {
      counts[char_to_idx(dna[i])]++;
  }
  if (check()) answer++;

  // 2. 슬라이딩 (P ~ S-1)
  for (int i = P_len; i < S_len; ++i) {
      counts[char_to_idx(dna[i - P_len])]--; // 맨 앞 문자 제거
      counts[char_to_idx(dna[i])]++;         // 맨 뒤 문자 추가
      if (check()) answer++;
  }
