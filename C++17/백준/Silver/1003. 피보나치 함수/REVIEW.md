## 한 줄 결론
- (이 문제는) **DP(메모이제이션)** / 복잡도: O(N_max + T) / `push_back`에 의존하는 미묘한 DP 구현

## 더 빠른/더 안전한 풀이(필수)
- **아이디어**: 0과 1의 호출 횟수 자체도 피보나치 점화식을 따르므로, 이 둘을 직접 바텀업 DP로 계산합니다.
- **복잡도**: O(N_max) 전처리 후, 각 테스트 케이스 O(1)
- **왜 이게 더 좋은지**: 재귀 오버헤드가 없고, 문제의 요구사항을 직접 계산하여 코드가 더 명확하고 안전합니다.

## 내가 지금 틀리기 쉬운 포인트 TOP3
- 1) **초기값/시작점/종료조건**: `N=0`일 때 `{1, 0}`, `N=1`일 때 `{0, 1}`이 되는 초기 DP 값 설정이 핵심입니다. 현재 코드는 `main`에서 `if(n==0)`으로 잘 처리했지만, DP 로직에 녹여내는 것이 더 일반적입니다.
- 2) **인덱스/경계**: `main`에서 `n=0`을 분기 처리하지 않았다면 `fibo(n-1)` 호출 시 음수 인덱스(-1)에 접근하려는 로직으로 이어져 런타임 에러가 발생했을 것입니다.
- 3) **자료구조/성능 함정**: 전역 `vector`에 `push_back`으로 DP 값을 채우는 방식은 재귀 호출 순서가 항상 `N`을 오름차순으로 채운다는 암묵적 가정에 의존합니다. `dp[i] = ...` 형태의 인덱스 기반 접근이 더 안전하고 명시적입니다.

## 즉시 수정 액션(최대 4개)
- **바텀업(Bottom-up) DP로 변경**: 재귀 대신 `for` 루프로 DP 테이블을 채우는 것이 더 직관적이고 스택 오버플로 위험이 없습니다.
- **전용 DP 테이블 사용**: 피보나치 수 대신, `{0 호출 횟수, 1 호출 횟수}`를 담는 `pair<int, int> dp[41]` 같은 자료구조를 사용하는 것이 문제 의도에 더 부합합니다.
- **전처리**: `main` 함수 시작 시 N의 최대값(40)까지의 답을 미리 계산해두면, 여러 테스트 케이스를 O(1)에 처리할 수 있습니다.
- `endl` 대신 `'\n'` 사용: `cout << ... << endl;`은 버퍼를 비우므로, `cout << ... << '\n';`이 경쟁 프로그래밍에선 더 선호됩니다.

```REVIEW_EXAMPLE
# 제목: 바텀업 DP를 이용한 더 직관적인 풀이
- 적용 조건: 재귀 호출의 스택 오버헤드를 피하고 코드를 더 명확하게 만들고 싶을 때.
- 핵심 아이디어: 0과 1의 호출 횟수에 대한 점화식을 직접 사용합니다.
  - `dp[i] = { i번째 피보나치 수를 구할 때 0과 1의 호출 횟수 }`
  - `dp[i].zeros = dp[i-1].zeros + dp[i-2].zeros`
  - `dp[i].ones = dp[i-1].ones + dp[i-2].ones`
- 복잡도: O(N_max) (전처리) + O(T) (쿼리 처리)
- 구현 스케치:
pair<int, int> dp[41];

void precompute() {
    dp[0] = {1, 0};
    dp[1] = {0, 1};
    for (int i = 2; i <= 40; ++i) {
        dp[i].first = dp[i-1].first + dp[i-2].first;
        dp[i].second = dp[i-1].second + dp[i-2].second;
    }
}

int main() {
    // ... 입출력 설정 ...
    precompute();
    int t;
    cin >> t;
    while (t--) {
        int n;
        cin >> n;
        cout << dp[n].first << " " << dp[n].second << '\n';
    }
}
```
