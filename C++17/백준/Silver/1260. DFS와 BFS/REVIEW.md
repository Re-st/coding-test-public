## 내가 지금 틀리기 쉬운 포인트 TOP3
- 1) 초기값/시작점/종료조건: **탐색 호출 순서**가 출력 순서와 다릅니다. DFS 결과를 먼저 출력해야 하므로 `main`에서 `dfs(v)`를 먼저 호출해야 합니다. 현재 코드는 BFS를 먼저 실행하여 순서가 맞지 않습니다.
- 2) 인덱스/경계/오버플로/방문처리: `dfs`와 `bfs` 사이에 `visited` 배열을 초기화하는 것을 잊기 쉽습니다. 현재 `memset`으로 잘 처리했지만, 누락 시 두 번째 탐색이 동작하지 않아 치명적입니다.
- 3) 자료구조/성능 함정: "정점 번호가 작은 것을 먼저 방문" 조건 때문에 인접 리스트 정렬이 필수입니다. 정렬을 빠뜨리면 입력 순서에 따라 결과가 달라져 오답 처리됩니다.

## 즉시 수정 액션(최대 4개)
- `main` 함수에서 `dfs(v)` 호출이 `bfs(v)` 호출보다 먼저 오도록 순서를 바꾸세요.
- C++에서는 `<string.h>` 대신 `<cstring>`을 사용하는 것이 표준입니다.
- 가독성을 위해 `for` 루프를 range-based for loop으로 개선할 수 있습니다.
  ```cpp
  // for (int i = 0; i < edges[x].size(); i++) { ... edges[x][i] ... }
  for (int next_node : edges[x]) {
      if (!visited[next_node]) {
          // ...
      }
  }
  ```
- (선택) 입출력량이 많을 경우를 대비해 `main` 상단에 `ios_base::sync_with_stdio(false); cin.tie(NULL);`를 추가하는 것을 권장합니다.
